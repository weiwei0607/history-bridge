# History Bridge 優化建議清單

## 🔴 必修（影響功能）

### 1. GameMode.jsx 有多餘括弧
- 第 214 行附近出現縮進錯亂 + 多餘的 `)`
- 可能導致編譯失敗或執行錯誤
- **修法**：清理尾端結構，確保 JSX 括弧對齊

### 2. 遊戲題目生成可能無限迴圈
- `gameLogic.js` 的 `generateQuestion()` 靠隨機抽樣找合法題目
- 如果資料量少、圖太稀疏，while 迴圈可能永遠找不到合法答案
- **修法**：加迴圈次數上限，超過就 fallback 到預設題

---

## 🟡 中等優先（影響體驗）

### 3. 圖譜建構邏輯重複
- `bfs.js` 和 `gameLogic.js` 各自建構一份圖，邏輯雷同
- 資料結構微妙差異導致維護困難
- **修法**：抽出 `buildGraph()` 共用函數，兩邊 import 同一份

### 4. 年份格式不一致
- `PathTimeline` 有 `yearLabel()` 做 BCE/CE 格式化
- `PersonModal` 直接字串拼接，沒有處理負數（西元前）
- **修法**：把 `yearLabel()` 移到 utils，全部統一呼叫

### 5. 搜尋框每次都遍歷全部人物
- `SearchBox.jsx` 每次 query 改變就重新 filter 所有人物
- 人物超過幾百筆時會有感延遲
- **修法**：用 `useMemo` 預先建立搜尋索引

### 6. 沒有 Error Boundary
- 若 `FIGURES` 或 `CONNECTIONS` 資料異常，整個 app 會白屏
- **修法**：在 App 層加 React Error Boundary，顯示友善錯誤訊息

### 7. URL 分享解析沒有錯誤提示
- 分享連結帶的 `figureA`/`figureB` 若無效，靜默忽略、不提示用戶
- **修法**：解析失敗時顯示 toast 或說明文字

---

## 🟢 輕微（polish 方向）

### 8. 遊戲難度固定，無法調整
- `GameMode` 固定 15 秒，`GuessWhoMode` 提示順序固定
- **改進方向**：加難度選項（簡單/普通/困難），影響倒數時間或提示數量

### 9. PersonModal 的相關人物只有直接連線
- 只顯示直接相鄰的人，沒有距離感
- **改進方向**：可標注「直接相識」vs「間接（2度）」，或加「找最短路徑」按鈕

### 10. HubFigures 缺乏說明
- 樞紐人物的中心性分數對一般使用者不直觀
- **改進方向**：加一句話解釋「為什麼這個人是橋梁」（連接了哪些時代/區域）

### 11. 快速建議人物對硬編碼
- App.jsx 裡的 quickSuggestions 是固定幾組
- **改進方向**：每次隨機從中心性最高的人物配對中抽取

### 12. 無使用行為追蹤
- 不知道哪些人物對最常被搜尋、哪些連線最受歡迎
- **改進方向**：接 Firebase Analytics 或簡單的 localStorage 計數
